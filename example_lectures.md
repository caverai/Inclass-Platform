# Example Lectures and Activities for InClass Platform

This document contains two example lectures and their corresponding demo activities for the InClass Platform. You can copy and paste the activity details directly into the instructor dashboard.

---

## Example 1: Introduction to Data Structures in Python

### Lecture Overview
This lecture introduces students to core Python data structures, focusing on Lists and Dictionaries. Students learn how to store, access, and manipulate collections of data efficiently.

### Demo Activity Details

**Activity No:** 1
**Title:** Mastering Python Lists and Dictionaries
**Activity Text (Description):**
In this activity, you will demonstrate your understanding of Python's most common collection types. Your task is to explain how to create a list of items, how to access elements using their index, and how to use a dictionary to store key-value pairs for more structured data. Imagine you are building a simple inventory system for a store.

**Objectives:**
1. "Define a list and explain how to add new elements to it using the append method."
2. "Explain how to access a specific element in a list using its zero-based index."
3. "Describe the structure of a dictionary as a collection of unique keys and their associated values."
4. "Show how to retrieve a value from a dictionary using its key."

### Activity 2: List Comprehensions and Sets
**Activity No:** 2
**Title:** Efficient Data with Comprehensions and Sets
**Activity Text (Description):**
Now that you know the basics, let's look at more advanced ways to handle data. Explain what a 'list comprehension' is and how it provides a shorter syntax for creating a new list from an existing one. Also, describe the 'Set' data type—how is it different from a list, and when would you use it to store unique items?

**Objectives:**
1. "Explain the syntax of a list comprehension: [expression for item in iterable]."
2. "Describe the primary benefit of list comprehensions: concise and readable code."
3. "Define a Set and highlight that it only contains unique elements."
4. "Identify a use case for Sets, such as removing duplicates from a list."

## Example 2: Fundamentals of Software Testing

### Lecture Overview
This lecture covers the importance of software testing in the development lifecycle. It focuses on Unit Testing, the process of testing individual components of a program to ensure they work as expected.

### Demo Activity Details

**Activity No:** 2
**Title:** The Art of Unit Testing
**Activity Text (Description):**
Welcome to the testing lab! In this activity, you will explore the basics of unit testing. You need to explain what a unit test is, why we write them, and how they help prevent regressions in our code. Discuss the concept of an 'assertion' and why it is the core of every test case.

**Objectives:**
1. "Define a unit test as a way to verify the smallest testable parts of an application in isolation."
2. "Explain the benefit of catching bugs early in the development cycle through automated tests."
3. "Describe the role of an assertion in determining whether a test passes or fails."
4. "Identify the difference between a test case and a test suite."

### Activity 2: Test-Driven Development (TDD)
**Activity No:** 4
**Title:** The Red-Green-Refactor Cycle
**Activity Text (Description):**
Test-Driven Development (TDD) is a powerful methodology. Explain the three main phases: Red, Green, and Refactor. Why do we write the test *before* the actual code? Discuss how this approach influences the design of your software.

**Objectives:**
1. "Explain the 'Red' phase: writing a test that fails because the feature doesn't exist yet."
2. "Explain the 'Green' phase: writing the minimum code necessary to make the test pass."
3. "Explain the 'Refactor' phase: cleaning up the code while keeping the tests passing."
4. "Discuss how TDD leads to better decoupled and more testable code designs."

## Instructions for Use

1. **Log in** as an Instructor.
2. Select a **Course**.
3. Click on **Create Activity**.
4. Copy the **Activity No**, **Title**, and **Activity Text** from the examples above.
5. Add the **Objectives** one by one in the objectives list.
6. Save and then **Start** the activity to make it available to students.
